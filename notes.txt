WHAT WORKS:

1-FFPLAY RTSP streaming from socket
	Terminal1 (host1, ip1): 
		# Write video to socket to simulates facet ucs:  
		$ nc -l -p 9090 < test.mp4

	Terminal2 (host2, ip2): 
		# Read video from term1 socket and start RTSP server
		$ cvlc -vvv tcp://<IP1>:9090 --sout '#rtp{sdp=rtsp://localhost:8554/testfeed}' 

	Terminal3 (host2, ip2):
		$ ffplay rtsp://localhost:8554/testfeed

2-BROWSER RTSP streaming from socket
	Terminal1 (host1, ip1): 
		# Write video to socket to simulates facet ucs:  
		$ nc -l -p 9090 < test.mp4

	Terminal2 (host2, ip2): 
		# Read video from term1 socket and start RTSP server
		$ cvlc -vvv tcp://<IP1>:9090 --sout '#rtp{sdp=rtsp://localhost:8554/testfeed}' 

	Terminal3 (host2, ip2):
		# Install Kurento as in http://doc-kurento.readthedocs.io/en/stable/installation_guide.html
		# Start Kurento server
		$ sudo service kurento-media-server-6.0 start

		# Install Kurento webrtc
		$ git clone https://github.com/lulop-k/kurento-rtsp2webrtc

		# Start an http server here and launch the demo app
		$ cd kurento-rtsp2webrtc
		$ google-chrome http://localhost:8080

		# Then play video in the web app by using this source URL: "rtsp://localhost:8554/testfeed"
